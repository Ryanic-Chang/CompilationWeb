# 编译原理前端实验平台 (Compilation Web) 开发规范

在为本项目开发新功能或实验模块时，请务必遵循以下规范：

## 1. 架构与技术栈
- **纯前端项目**：仅使用 HTML5, 原生 JavaScript (ES6+), 和 CSS。
- **样式框架**：使用通过 CDN 引入的 `Tailwind CSS` 进行现代化、明亮风格的样式设计。
- **图形库**：使用 `vis-network` 进行状态图或树状图的可视化展示（如果需要绘图）。
- **无构建工具**：不使用 Webpack/Vite/Node.js 等构建工具。确保所有代码都可以直接在浏览器中通过双击 `index.html` 运行。
- **第三方库引入**：只使用 CDN 引入，切勿使用 `npm install`。

## 2. UI与交互规范
- **导航结构**：左侧为多级导航侧边栏 (`aside` -> `nav`)，右侧为实验内容展示区 (`main`)。
- **页面切换**：所有实验模块都写在同一个 `index.html` 中，通过 `.page-section` 类的显隐 (`active` 类) 来切换页面。全局交互逻辑由 `app.js` 统一控制。
- **视觉风格一致性**：
  - 页面背景以 `bg-slate-50` 为主。
  - 卡片面板使用白底 (`bg-white`) 加阴影 (`shadow-sm`) 和边框 (`border-slate-200`)。
  - 按钮和主色调使用 `indigo-600`。
- **输入设计**：尽量避免让用户手动输入格式化的长文本。应设计结构化的表单输入，并提供文件上传自动解析回填功能。
- **状态反馈**：涉及解析、验证等操作时，必须在 UI 上提供明确的成功（绿色，如 `text-emerald-700 bg-emerald-50`）或失败（红色，如 `text-red-600 bg-red-50`）提示。

## 3. 新实验模块开发流程
当你被要求添加一个新的实验（如“实验二：LL(1)”）时，请按照以下步骤执行：

1. **更新导航栏**：
   - 在 `index.html` 的 `<nav>` 中添加新的导航项，指定唯一的 `data-target`（例如 `experiment-ll1`）。
   - 移除原有的 `(待开发)` 占位符。

2. **新增页面结构**：
   - 在 `index.html` 的 `<main>` 中新增对应的 `<section id="experiment-ll1" class="page-section max-w-7xl mx-auto hidden">` 块。
   - 包含左侧控制面板（输入、测试）和右侧结果展示区（可视化或表格）。

3. **编写独立业务逻辑**：
   - 创建对应的业务逻辑脚本文件（如 `ll1.js`），并在 `index.html` 底部引入。
   - 在 JS 文件中，保持面向对象的编写方式（如 `class LL1 { ... }`）。
   - 明确分离“核心算法类”和“UI DOM 交互逻辑”。
   - 添加中文注释以保证代码可读性。

4. **容错处理**：
   - 对于所有用户输入和文件解析，务必包含完善的合法性检查，防止程序崩溃。
